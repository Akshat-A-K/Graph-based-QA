Problem Statement and Objective

The project solves document question answering using multi-level graph reasoning. The objective is to answer bridge and comparison questions with supporting evidence and measurable quality.

Challenges in Document QA

  • Long context and multiple entities
  • Multi-hop reasoning between facts
  • Comparison questions need structured logic
  • Need explainable evidence, not only final answer

Project Objective

  • Build DRG, Span Graph, and Knowledge Graph
  • Use hybrid retrieval for stronger evidence selection
  • Return answer with confidence and supporting spans
  • Evaluate on HotpotQA with EM and F1

System Architecture and Core Components

Input and Parsing

Enhanced Reasoner and Answer Retrieval

Query understanding, hybrid retrieval, graph reasoning, and extractive answer selection in one flow.

Enhanced Hybrid Reasoner

  • Hybrid scoring: semantic + BM25 + centrality.
  • Centrality: PageRank and betweenness for node importance.
  • Traversal: depth-based expansion with discourse-priority neighbors.
  • Query expansion: transformer-driven span expansions.
  • KG-guided retrieval: query entities -> KG neighbors -> candidate spans.
  • Cross-encoder: optional re-ranking for precision refinement.

Answer Selector Pipeline

  • Merge candidates from final spans, hybrid, traversal, expansion, and KG results.
  • Score using overlap, sentence preference, and concise-length penalties.
  • Deduplicate by sentence to improve evidence diversity.
  • Run extractive QA on top contexts and fallback contexts.
  • Use BM25 global sentence fallback when QA confidence is low.
  • Return final answer + supporting evidence + combined confidence.

Result Insights from TXT and JSON Outputs

Strengths Observed

1) Full answer coverage: 500/500 answered.
2) Bridge questions perform better than comparison overall.
3) Easy level shows strongest EM and F1.
4) KG evidence and graph-based retrieval provide stable evidence count.

Error Patterns Observed

1) Normalization mismatch (example: "Kazakhstan" vs "Republic of Kazakhstan").
2) Comparison questions often return entity phrase instead of boolean/common relation.
3) Hard level multi-hop chains still difficult for precise extraction.
4) Some answers are near-correct but lose EM due to extra words/punctuation.

Final Interpretation

The implemented pipeline is robust in coverage and evidence retrieval, with strong bridge-question handling. Main improvement area is fine-grained answer normalization and comparison reasoning quality, especially for hard questions.
